Introduction to SQL

SQL (Structured Query Language) is the standard language for managing and manipulating relational databases. It was developed by IBM in the 1970s and became an ANSI/ISO standard in 1986.

DDL

Data Definition Language — defines schema (CREATE, ALTER, DROP)

DML

Data Manipulation Language — manipulates data (INSERT, UPDATE, DELETE)

DQL

Data Query Language — retrieves data (SELECT)

DCL

Data Control Language — permissions (GRANT, REVOKE)

TCL

Transaction Control Language (COMMIT, ROLLBACK, SAVEPOINT)

Key PointSQL is case-insensitive for keywords, but it's a best practice to write keywords in UPPERCASE for readability.

03

DDL — Data Definition Language

CREATE TABLE

Used to create a new table in the database.

CREATE TABLE students (
    student_id   INT            PRIMARY KEY AUTO_INCREMENT,
    name         VARCHAR(100)   NOT NULL,
    email        VARCHAR(150)   UNIQUE,
    age          INT            CHECK (age >= 18),
    dept_id      INT,
    enrolled_on  DATE           DEFAULT CURRENT_DATE,
    FOREIGN KEY (dept_id) REFERENCES departments(dept_id)
);

ALTER TABLE

Used to add, modify, or drop columns from an existing table.

-- Add a column
ALTER TABLE students ADD COLUMN phone VARCHAR(15);

-- Modify a column
ALTER TABLE students MODIFY COLUMN name VARCHAR(200) NOT NULL;

-- Drop a column
ALTER TABLE students DROP COLUMN phone;

-- Rename a column
ALTER TABLE students RENAME COLUMN name TO full_name;

-- Rename table
ALTER TABLE students RENAME TO learners;

DROP & TRUNCATE

-- Drop table permanently (removes structure + data)
DROP TABLE students;

-- Truncate (removes all data, keeps structure, faster than DELETE)
TRUNCATE TABLE students;

-- Drop database
DROP DATABASE mydb;

-- Create database
CREATE DATABASE mydb;
USE mydb;
WarningDROP and TRUNCATE are irreversible by default. Always take a backup before running these commands in production.

DML — Data Manipulation Language

INSERT

-- Insert single row
INSERT INTO students (name, email, age, dept_id)
VALUES ('Sameeksha', 'sameeksha@email.com', 21, 3);

-- Insert multiple rows
INSERT INTO students (name, email, age) VALUES
    ('Ananya', 'a@email.com', 22),
    ('Rohan',  'r@email.com', 20),
    ('Priya',  'p@email.com', 23);

-- Insert from another table
INSERT INTO archive_students
SELECT * FROM students WHERE age > 25;

UPDATE

-- Update a single row
UPDATE students
SET   email = 'new@email.com', age = 22
WHERE student_id = 1;

-- Update multiple rows
UPDATE students
SET   dept_id = 5
WHERE dept_id IS NULL;

DELETE

-- Delete specific rows
DELETE FROM students
WHERE student_id = 10;

-- Delete all rows (keep structure)
DELETE FROM students;

DQL — SELECT Statement

Basic SELECT Syntax

SELECT   column1, column2, ...
FROM     table_name
WHERE    condition
GROUP BY column
HAVING   group_condition
ORDER BY column [ASC|DESC]
LIMIT    n
OFFSET   m;

SELECT Examples

-- Select all columns
SELECT * FROM students;

-- Select specific columns with alias
SELECT name AS student_name, age, email
FROM students;

-- DISTINCT — removes duplicates
SELECT DISTINCT dept_id FROM students;

-- ORDER BY
SELECT * FROM students
ORDER BY age DESC, name ASC;

-- LIMIT and OFFSET (pagination)
SELECT * FROM students
ORDER BY student_id
LIMIT 10 OFFSET 20;  -- page 3 (0-indexed)

-- Computed columns
SELECT name, salary, salary * 1.1 AS new_salary
FROM employees;

WHERE Clause & Operators

Comparison Operators

OperatorMeaningExample
=EqualWHERE age = 21
!= or <>Not equalWHERE age <> 21
>Greater thanWHERE age > 18
<Less thanWHERE salary < 50000
>=Greater or equalWHERE marks >= 60
<=Less or equalWHERE age <= 30

Special Operators

-- BETWEEN (inclusive range)
SELECT * FROM students
WHERE age BETWEEN 18 AND 25;

-- IN (list of values)
SELECT * FROM students
WHERE dept_id IN (1, 2, 5);

-- NOT IN
SELECT * FROM students
WHERE dept_id NOT IN (3, 4);

-- IS NULL / IS NOT NULL
SELECT * FROM students WHERE email IS NULL;
SELECT * FROM students WHERE email IS NOT NULL;

-- LIKE (pattern matching)
SELECT * FROM students WHERE name LIKE 'S%';     -- starts with S
SELECT * FROM students WHERE name LIKE '%sha';   -- ends with sha
SELECT * FROM students WHERE name LIKE '%mee%';  -- contains mee
SELECT * FROM students WHERE name LIKE 'S_____a';-- _ matches 1 char

-- AND / OR / NOT
SELECT * FROM students
WHERE (age > 20 AND dept_id = 3)
   OR (age < 18 AND NOT email IS NULL);
Tip — LIKE wildcards% matches zero or more characters. _ matches exactly one character. Use ILIKE in PostgreSQL for case-insensitive matching.

SQL Joins

Joins combine rows from two or more tables based on a related column.

INNER JOIN

Returns rows that have matching values in both tables.

LEFT JOIN

All rows from left table + matching from right (NULLs if no match).

RIGHT JOIN

All rows from right table + matching from left.

FULL OUTER JOIN

All rows from both tables; NULLs for non-matches.

CROSS JOIN

Cartesian product — every combination of rows.

SELF JOIN

A table joined with itself using aliases.

INNER JOIN

SELECT s.name, s.age, d.dept_name
FROM   students s
INNER JOIN departments d ON s.dept_id = d.dept_id;

LEFT JOIN

-- All students including those with no department
SELECT s.name, d.dept_name
FROM   students s
LEFT JOIN departments d ON s.dept_id = d.dept_id;

RIGHT JOIN

-- All departments including those with no students
SELECT s.name, d.dept_name
FROM   students s
RIGHT JOIN departments d ON s.dept_id = d.dept_id;

FULL OUTER JOIN

SELECT s.name, d.dept_name
FROM   students s
FULL OUTER JOIN departments d ON s.dept_id = d.dept_id;

SELF JOIN

-- Find employees and their managers
SELECT e.name AS employee, m.name AS manager
FROM   employees e
LEFT JOIN employees m ON e.manager_id = m.emp_id;

CROSS JOIN

-- All size-color combinations
SELECT s.size_name, c.color_name
FROM   sizes s
CROSS JOIN colors c;

Aggregate Functions

Aggregate functions perform a calculation on a set of values and return a single value.

FunctionDescriptionExample
COUNT()Number of rowsCOUNT(*) or COUNT(col)
SUM()Total of a numeric columnSUM(salary)
AVG()Average valueAVG(marks)
MAX()Maximum valueMAX(age)
MIN()Minimum valueMIN(price)
 
SELECT
    COUNT(*)           AS total_students,
    COUNT(email)       AS students_with_email,
    AVG(age)           AS avg_age,
    MAX(age)           AS oldest,
    MIN(age)           AS youngest,
    SUM(scholarship)   AS total_scholarship
FROM students;
NoteCOUNT(*) counts all rows including NULLs. COUNT(column) counts only non-NULL values in that column.

GROUP BY & HAVING

GROUP BY

Groups rows with the same values in specified columns and applies aggregate functions to each group.

-- Count students per department
SELECT   dept_id, COUNT(*) AS student_count
FROM     students
GROUP BY dept_id
ORDER BY student_count DESC;

-- Multiple columns in GROUP BY
SELECT   dept_id, gender, AVG(age) AS avg_age
FROM     students
GROUP BY dept_id, gender;

HAVING

HAVING filters groups after GROUP BY. Use it where you can't use WHERE (i.e., on aggregate functions).

-- Departments with more than 10 students
SELECT   dept_id, COUNT(*) AS count
FROM     students
GROUP BY dept_id
HAVING   COUNT(*) > 10;

-- WHERE vs HAVING — used together
SELECT   dept_id, AVG(age) AS avg_age
FROM     students
WHERE    age >= 18              -- filter rows first
GROUP BY dept_id
HAVING   AVG(age) > 21;         -- filter groups
WHERE vs HAVINGWHERE filters individual rows before grouping. HAVING filters groups after GROUP BY. You cannot use aggregate functions in WHERE.

Subqueries

A subquery is a query nested inside another query. It can appear in SELECT, FROM, or WHERE clauses.

Subquery in WHERE

-- Students older than the average age
SELECT name, age
FROM   students
WHERE  age > (SELECT AVG(age) FROM students);

-- Students in departments located in Bangalore
SELECT name
FROM   students
WHERE  dept_id IN (
    SELECT dept_id FROM departments
    WHERE  location = 'Bangalore'
);

Subquery in FROM (Derived Table)

SELECT dept_id, avg_salary
FROM (
    SELECT dept_id, AVG(salary) AS avg_salary
    FROM   employees
    GROUP BY dept_id
) AS dept_avg
WHERE avg_salary > 60000;

Correlated Subquery

-- Employees earning more than their dept average
SELECT name, salary, dept_id
FROM   employees e
WHERE  salary > (
    SELECT AVG(salary)
    FROM   employees
    WHERE  dept_id = e.dept_id  -- references outer query
);

EXISTS / NOT EXISTS

-- Departments that have at least one student
SELECT dept_name
FROM   departments d
WHERE EXISTS (
    SELECT 1
    FROM   students s
    WHERE  s.dept_id = d.dept_id
);

Views

A view is a virtual table based on the result of a SELECT query. It does not store data itself.

-- Create a view
CREATE VIEW student_dept_view AS
SELECT s.name, s.age, d.dept_name
FROM   students s
JOIN   departments d ON s.dept_id = d.dept_id;

-- Use the view like a table
SELECT * FROM student_dept_view
WHERE dept_name = 'Computer Science';

-- Update a view
CREATE OR REPLACE VIEW student_dept_view AS
SELECT s.name, s.email, d.dept_name
FROM   students s
JOIN   departments d ON s.dept_id = d.dept_id;

-- Drop a view
DROP VIEW student_dept_view;
Benefits of ViewsSimplify complex queries, provide security by hiding sensitive columns, present consistent data representation to users.
12

Indexes

An index speeds up data retrieval. It's created on columns frequently used in WHERE, JOIN, or ORDER BY.

-- Create an index
CREATE INDEX idx_student_name
ON students (name);

-- Unique index
CREATE UNIQUE INDEX idx_email
ON students (email);

-- Composite index (multiple columns)
CREATE INDEX idx_dept_age
ON students (dept_id, age);

-- Drop an index
DROP INDEX idx_student_name;

-- View indexes on a table (MySQL)
SHOW INDEXES FROM students;
CautionIndexes speed up SELECT but slow down INSERT, UPDATE, DELETE operations because the index must be maintained. Don't over-index.

CREATE TABLE orders (

Transactions (TCL)

A transaction is a sequence of SQL statements treated as a single unit. It follows the ACID properties.

Atomicity

All operations succeed or all are rolled back.

Consistency

DB moves from one valid state to another.

Isolation

Transactions don't interfere with each other.

Durability

Committed changes are permanently saved.

START TRANSACTION;  -- or BEGIN;

UPDATE accounts SET balance = balance - 5000
WHERE account_id = 101;

UPDATE accounts SET balance = balance + 5000
WHERE account_id = 202;

COMMIT;   -- Save changes permanently

-- Or rollback on error
ROLLBACK;

-- SAVEPOINT — partial rollback
SAVEPOINT sp1;
-- ... do some operations ...
ROLLBACK TO SAVEPOINT sp1;
RELEASE SAVEPOINT sp1;

Stored Procedures

A stored procedure is a prepared SQL code that can be saved and reused. It can accept parameters and perform complex logic.

-- Create a stored procedure
DELIMITER $$

CREATE PROCEDURE GetStudentsByDept(
    IN  dept_input  INT,
    OUT total_count INT
)
BEGIN
    SELECT name, age, email
    FROM   students
    WHERE  dept_id = dept_input;

    SELECT COUNT(*) INTO total_count
    FROM   students
    WHERE  dept_id = dept_input;
END$$

DELIMITER ;

-- Call the procedure
CALL GetStudentsByDept(3, @count);
SELECT @count;

-- Drop procedure
DROP PROCEDURE IF EXISTS GetStudentsByDept;

Triggers

A trigger is a stored procedure that automatically executes in response to certain events (INSERT, UPDATE, DELETE) on a table.

DELIMITER $$

CREATE TRIGGER before_student_insert
BEFORE INSERT ON students
FOR EACH ROW
BEGIN
    -- Auto-capitalize name before insert
    SET NEW.name = UPPER(NEW.name);
END$$

-- AFTER UPDATE trigger — log changes
CREATE TRIGGER after_salary_update
AFTER UPDATE ON employees
FOR EACH ROW
BEGIN
    IF OLD.salary != NEW.salary THEN
        INSERT INTO salary_audit
        (emp_id, old_salary, new_salary, changed_at)
        VALUES
        (NEW.emp_id, OLD.salary, NEW.salary, NOW());
    END IF;
END$$

DELIMITER ;

-- Drop a trigger
DROP TRIGGER IF EXISTS before_student_insert;

Normalization

Normalization is the process of organizing data to reduce redundancy and improve integrity.

Normal FormRule
1NFAtomic values only — no repeating groups. Each column has a single value.
2NF1NF + No partial dependency. All non-key columns depend on the full primary key.
3NF2NF + No transitive dependency. Non-key columns depend only on the primary key.
BCNF3NF + Every determinant must be a candidate key (stronger version of 3NF).
4NFBCNF + No multi-valued dependencies.
5NF4NF + No join dependencies — the table cannot be decomposed further.
Key TermsFunctional Dependency (A → B: knowing A determines B). Partial Dependency: non-key depends on part of composite key. Transitive Dependency: A → B → C (C depends on A through B).

Window Functions

Window functions perform calculations across a set of rows related to the current row — without collapsing them into a single output row like GROUP BY does.

-- Syntax: function() OVER (PARTITION BY col ORDER BY col)

-- ROW_NUMBER — unique sequential row number per partition
SELECT name, dept_id, salary,
       ROW_NUMBER() OVER (
           PARTITION BY dept_id
           ORDER BY salary DESC
       ) AS row_num
FROM employees;

-- RANK — same rank for ties, gaps in sequence
RANK() OVER (ORDER BY salary DESC)

-- DENSE_RANK — same rank for ties, no gaps
DENSE_RANK() OVER (ORDER BY salary DESC)

-- NTILE — divide into n buckets
NTILE(4) OVER (ORDER BY salary)  -- quartiles

-- LAG — access previous row value
LAG(salary, 1, 0) OVER (ORDER BY hire_date)

-- LEAD — access next row value
LEAD(salary, 1, 0) OVER (ORDER BY hire_date)

-- Running total with SUM OVER
SELECT name, salary,
       SUM(salary) OVER (ORDER BY hire_date
           ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS running_total
FROM employees;

Top N per Group (Common Interview Pattern)

-- Top 3 highest paid employees in each department
SELECT * FROM (
    SELECT name, dept_id, salary,
           DENSE_RANK() OVER (
               PARTITION BY dept_id
               ORDER BY salary DESC
           ) AS rnk
    FROM employees
) ranked
WHERE rnk <= 3;

Quick Reference Cheat Sheet

SQL Execution Order

Order of Execution (not writing order) FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT

SET Operations

-- UNION — combine results, remove duplicates
SELECT name FROM students
UNION
SELECT name FROM staff;

-- UNION ALL — keep duplicates
SELECT name FROM students
UNION ALL
SELECT name FROM staff;

-- INTERSECT — common rows
SELECT name FROM students
INTERSECT
SELECT name FROM staff;

-- EXCEPT / MINUS — rows in first not in second
SELECT name FROM students
EXCEPT
SELECT name FROM staff;

CTEs (Common Table Expressions)

-- WITH clause — cleaner than subqueries
WITH dept_avg AS (
    SELECT dept_id, AVG(salary) AS avg_sal
    FROM   employees
    GROUP BY dept_id
),
high_earners AS (
    SELECT e.name, e.salary, da.avg_sal
    FROM   employees e
    JOIN   dept_avg da ON e.dept_id = da.dept_id
    WHERE  e.salary > da.avg_sal
)
SELECT * FROM high_earners
ORDER BY salary DESC;

DCL — Permissions

-- Grant privileges
GRANT SELECT, INSERT ON students TO 'sameeksha'@'localhost';
GRANT ALL PRIVILEGES ON mydb.* TO 'admin'@'%';

-- Revoke privileges
REVOKE INSERT ON students FROM 'sameeksha'@'localhost';

-- Flush (apply changes)
FLUSH PRIVILEGES;
